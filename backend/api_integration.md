## 2. Аутентификация

API использует JWT-аутентификацию (Bearer token).

### Заголовок авторизации

```
Authorization: Bearer <access_token>
```

### Время жизни токенов

| Токен | Время жизни |
|---|---|
| `access` | 7 дней |
| `refresh` | 30 дней |

### Когда нужен токен

Большинство эндпоинтов публичны (`AllowAny`). Токен не является обязательным для получения меню или создания заказа. Передавайте токен, если хотите работать с профилем конкретного пользователя.

### Обновление access-токена

Когда `access`-токен истёк, используйте `refresh`-токен для получения нового:

```
POST /api/auth/token/refresh/
{ "refresh": "<refresh_token>" }
```

## 3. Эндпоинты авторизации

Префикс: `/api/auth/`

---

### 3.1 Запрос кода авторизации

```
POST /api/auth/request-code/
```

Генерирует 6-значный код для указанного номера телефона. В рабочей реализации код доставляется через Telegram-бота. Все предыдущие неиспользованные коды для этого номера автоматически инвалидируются.

**Тело запроса:**

```json
{
  "phone_number": "+79001234567"
}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `phone_number` | string | да | Номер телефона в любом формате |

**Ответ `200 OK`:**

```json
{
  "detail": "Код отправлен в Telegram. Введите его для входа."
}
```

**Ошибки:**

| Код | Причина |
|---|---|
| `400` | Не передан `phone_number` или невалидный формат |

---

### 3.2 Верификация кода и получение JWT

```
POST /api/auth/verify-code/
```

Проверяет код. При успехе помечает код как использованный, создаёт пользователя (если не существует), возвращает JWT-токены.

**Тело запроса:**

```json
{
  "phone_number": "+79001234567",
  "code": "482910"
}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `phone_number` | string | да | Тот же номер, что был передан в `request-code` |
| `code` | string | да | 6-значный код (строка, не число) |

**Ответ `200 OK`:**

```json
{
  "access": "<jwt_access_token>",
  "refresh": "<jwt_refresh_token>",
  "user": {
    "id": 1,
    "phone_number": "+79001234567",
    "name": "",
    "email": "",
    "date_joined": "2024-01-15T10:30:00Z",
    "addresses": []
  }
}
```

**Ошибки:**

| Код | Причина |
|---|---|
| `400` | Неверный или уже использованный код: `{"detail": "Неверный или уже использованный код."}` |
| `400` | Код истёк (10 минут): `{"detail": "Срок действия кода истёк."}` |

> **Нюанс:** Сохраните `user.id` из ответа — он понадобится для запросов к заказам (`/api/orders/users/{user_id}/`).

---

### 3.3 Обновление access-токена

```
POST /api/auth/token/refresh/
```

**Тело запроса:**

```json
{
  "refresh": "<jwt_refresh_token>"
}
```

**Ответ `200 OK`:**

```json
{
  "access": "<new_jwt_access_token>"
}
```

**Ошибки:**

| Код | Причина |
|---|---|
| `401` | `refresh`-токен истёк или недействителен — требуется повторная авторизация |

---

### 3.4 Создание пользователя

```
POST /api/auth/users/
```

Прямое создание пользователя без кода авторизации. Используется в административных целях. Для клиентского flow предпочтительнее использовать `verify-code`.

**Тело запроса:**

```json
{
  "phone_number": "+79001234567",
  "name": "Иван",
  "email": "ivan@example.com"
}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `phone_number` | string | да | Уникальный идентификатор пользователя |
| `name` | string | нет | Имя пользователя |
| `email` | string | нет | Email пользователя |

**Ответ `201 Created`:**

```json
{
  "id": 1,
  "phone_number": "+79001234567",
  "name": "Иван",
  "email": "ivan@example.com",
  "date_joined": "2024-01-15T10:30:00Z",
  "addresses": []
}
```

---

### 3.5 Получение профиля пользователя

```
GET /api/auth/users/{id}/
```

Возвращает данные пользователя, включая все сохранённые адреса доставки.

**Ответ `200 OK`:**

```json
{
  "id": 1,
  "phone_number": "+79001234567",
  "name": "Иван",
  "email": "ivan@example.com",
  "date_joined": "2024-01-15T10:30:00Z",
  "addresses": [
    {
      "id": 1,
      "address": "ул. Пушкина, д. 1, кв. 5",
      "created_at": "2024-01-15T11:00:00Z"
    }
  ]
}
```

> **Нюанс:** Адреса в `addresses` — это история доставки. Адреса накапливаются автоматически при создании заказа, если передан новый адрес (используйте для подсказок при вводе адреса в форме).

## 4. Эндпоинты меню

Префикс: `/api/restaurant/`

---

### 4.1 Список категорий

```
GET /api/restaurant/categories/
```

**Ответ `200 OK`:**

```json
[
  {
    "id": 1,
    "name": "Пицца"
  },
  {
    "id": 2,
    "name": "Напитки"
  }
]
```

> Используйте `id` категории в качестве фильтра для `/api/restaurant/variants/?category=1`.

---

### 4.2 Список блюд с вариантами

```
GET /api/restaurant/variants/
```

Возвращает список блюд, сгруппированных по блюду. Каждое блюдо содержит массив своих вариантов. Фильтры применяются по вариантам — блюдо попадёт в выдачу, если хотя бы один его вариант соответствует фильтрам.

**Параметры фильтрации (query string):**

| Параметр | Тип | Описание |
|---|---|---|
| `category` | integer | id категории |
| `calories_min` | number | Минимальная калорийность |
| `calories_max` | number | Максимальная калорийность |
| `price_min` | number | Минимальная цена |
| `price_max` | number | Максимальная цена |

**Примеры:**
```
GET /api/restaurant/variants/?category=1
GET /api/restaurant/variants/?price_min=200&price_max=500
GET /api/restaurant/variants/?category=1&calories_max=1000
```

**Ответ `200 OK`:**

```json
[
  {
    "id": 1,
    "dish_name": "Маргарита",
    "dish_image": "http://localhost:8000/media/dishes/margarita.jpg",
    "dish_description": "Классическая пицца с томатным соусом и моцареллой.",
    "category": {
      "id": 1,
      "name": "Пицца"
    },
    "variants": [
      {
        "id": 1,
        "size": "Маленький",
        "size_value": "25 см",
        "weight": "380.00",
        "calories": "650.00",
        "price": "399.00"
      },
      {
        "id": 2,
        "size": "Большой",
        "size_value": "35 см",
        "weight": "560.00",
        "calories": "950.00",
        "price": "649.00"
      }
    ]
  }
]
```

**Структура объекта блюда:**

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | ID блюда |
| `dish_name` | string | Название блюда |
| `dish_image` | string \| null | Абсолютный URL изображения или `null` |
| `dish_description` | string | Описание блюда |
| `category` | object | Категория блюда |
| `variants` | array | Список вариантов блюда |

**Структура варианта (`variants[]`):**

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | ID варианта — используется при создании заказа |
| `size` | string | Название размера (например, «Маленький») |
| `size_value` | string | Значение размера с единицей измерения (например, «25 см») |
| `weight` | string | Вес в граммах (decimal строка) |
| `calories` | string | Калорийность (decimal строка) |
| `price` | string | Цена (decimal строка) |

> **Нюанс:** `dish_image` — это абсолютный URL, включающий хост. В dev-окружении это `http://localhost:8000/media/...`. Используйте его напрямую для `<img src>`.

> **Нюанс:** Поля `weight`, `calories`, `price` — строки с десятичным числом (например, `"399.00"`). Приводите к числу для отображения: `parseFloat("399.00")`.

> **Нюанс:** Если применён фильтр по цене/калорийности и у блюда несколько вариантов, в `variants[]` будут только те варианты, которые прошли фильтр — не все варианты блюда.

```

```

**Ответ `200 OK`:**

```json
{
  "id": 1,
  "dish": 1,
  "dish_name": "Маргарита",
  "dish_image": "http://localhost:8000/media/dishes/margarita.jpg",
  "dish_description": "Классическая пицца с томатным соусом и моцареллой.",
  "category": {
    "id": 1,
    "name": "Пицца"
  },
  "size": {
    "id": 1,
    "label": "Маленький"
  },
  "size_value": "25.00",
  "measure_unit": {
    "id": 1,
    "label": "Сантиметры",
    "short": "см"
  },
  "weight": "380.00",
  "calories": "650.00",
  "price": "399.00"
}
```

> **Нюанс:** В детальном эндпоинте `size_value` — чистое число без единицы измерения (`"25.00"`), единица находится в `measure_unit.short`. В списочном эндпоинте они уже объединены в строку.

## 5. Эндпоинты заказов

Префикс: `/api/orders/`

---

### 5.1 Создание заказа

```
POST /api/orders/
```

Создаёт заказ. Пользователь идентифицируется по номеру телефона. Если пользователь с таким номером не существует — создаётся автоматически. Если адрес доставки новый для этого пользователя — сохраняется в его историю адресов.

**Тело запроса:**

```json
{
  "phone_number": "+79001234567",
  "name": "Иван",
  "email": "ivan@example.com",
  "address": "ул. Пушкина, д. 1, кв. 5",
  "comment": "Позвонить за 10 минут",
  "items": [
    { "dish_variant_id": 1, "quantity": 2 },
    { "dish_variant_id": 3, "quantity": 1 }
  ]
}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `phone_number` | string | **да** | Идентификатор пользователя |
| `name` | string | нет | Имя (заполняется если у пользователя пустое) |
| `email` | string | нет | Email (заполняется если у пользователя пустой) |
| `address` | string | нет | Адрес доставки. Если пустой — заказ считается самовывозом |
| `comment` | string | нет | Комментарий к заказу |
| `items` | array | **да** | Список позиций, минимум 1 элемент |
| `items[].dish_variant_id` | integer | **да** | ID варианта блюда из `/api/restaurant/variants/` |
| `items[].quantity` | integer | **да** | Количество (минимум 1) |

**Ответ `201 Created`:**

```json
{
  "id": 42,
  "user": 1,
  "address": "ул. Пушкина, д. 1, кв. 5",
  "started_at": "2024-01-15T12:00:00Z",
  "finished_at": null,
  "comment": "Позвонить за 10 минут",
  "status": "processing",
  "status_display": "В обработке",
  "is_pickup": false,
  "items": [
    {
      "id": 101,
      "dish_variant": {
        "id": 1,
        "dish": 1,
        "dish_name": "Маргарита",
        "dish_image": "http://localhost:8000/media/dishes/margarita.jpg",
        "dish_description": "...",
        "category": { "id": 1, "name": "Пицца" },
        "size": { "id": 1, "label": "Маленький" },
        "size_value": "25.00",
        "measure_unit": { "id": 1, "label": "Сантиметры", "short": "см" },
        "weight": "380.00",
        "calories": "650.00",
        "price": "399.00"
      },
      "quantity": 2
    }
  ]
}
```

**Структура ответа заказа:**

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | ID заказа |
| `user` | integer | ID пользователя |
| `address` | string | Адрес доставки (пустая строка при самовывозе) |
| `started_at` | string (ISO 8601) | Время создания заказа |
| `finished_at` | string \| null | Время завершения (null у нового заказа) |
| `comment` | string | Комментарий к заказу |
| `status` | string | Машинный статус (см. таблицу статусов) |
| `status_display` | string | Человекочитаемый статус |
| `is_pickup` | boolean | `true` если заказ на самовывоз (адрес пустой) |
| `items` | array | Позиции заказа с полными данными о варианте |

**Статусы заказа:**

| `status` | `status_display` | Описание |
|---|---|---|
| `processing` | В обработке | Начальный статус при создании |
| `preparing` | Готовится | Заказ принят и готовится |
| `courier_on_way` | Курьер в пути | Курьер выехал |
| `delivered` | Доставлен | Заказ доставлен |
| `cancelled` | Отменён | Заказ отменён |

**Ошибки:**

| Код | Причина |
|---|---|
| `400` | Не передан `phone_number` |
| `400` | Пустой массив `items`: `{"items": ["Заказ должен содержать хотя бы один элемент."]}` |
| `400` | Несуществующий или удалённый `dish_variant_id` |
| `400` | `quantity` < 1 |

## 6. Коды ошибок

| HTTP код | Общая причина |
|---|---|
| `400 Bad Request` | Ошибка валидации — тело ответа содержит описание полей с ошибками |
| `404 Not Found` | Объект не найден |
| `401 Unauthorized` | Токен отсутствует, истёк или недействителен |

**Формат ошибки валидации (400):**

```json
{
  "phone_number": ["Это поле обязательно."],
  "items": ["Заказ должен содержать хотя бы один элемент."]
}
```

**Формат общей ошибки:**

```json
{
  "detail": "Неверный или уже использованный код."
}
```

## 7. Важные нюансы

### Авторизация

- **Два шага:** сначала `request-code`, затем `verify-code`. Нельзя пропустить первый шаг.
- **Код действителен 10 минут.** После истечения нужно запросить новый.
- **Один активный код:** повторный вызов `request-code` для того же номера инвалидирует предыдущий код.
- **Автосоздание пользователя:** `verify-code` создаёт пользователя при первом входе. Отдельная регистрация не требуется.
- **`user.id` из `verify-code`** нужно сохранить в клиентском состоянии — он используется в URL-адресах заказов.

### Меню

- **Группировка по блюду:** `/api/restaurant/variants/` возвращает список блюд, а не вариантов. Каждое блюдо содержит `variants[]`.
- **Фильтрация отсекает варианты:** при фильтрации по цене/калорийности у блюда могут остаться не все варианты, а только прошедшие фильтр.
- **`dish_image` — абсолютный URL** с хостом сервера, использовать напрямую.
- **Удалённые блюда** (`is_deleted=true`) в API не возвращаются.
- **`dish_variant_id` в `variants[].id`** — используется при создании заказа.

### Заказы

- **Самовывоз:** передайте пустую строку `"address": ""` или не передавайте поле — `is_pickup` в ответе будет `true`.
- **Автосохранение адреса:** если адрес ещё не был у пользователя, он добавляется в `addresses` профиля. Удобно для подсказок при повторном заказе.
- **Обновление имени/email:** при создании заказа `name` и `email` заполняются только если у существующего пользователя эти поля пустые.
- **`items[].dish_variant` в ответе** содержит полный объект варианта блюда (плоский формат, как в `GET /api/restaurant/variants/{id}/`).
- **Начальный статус** нового заказа — всегда `processing`.
- **Смена статуса** происходит на стороне сервера/администратора — клиент только читает текущий статус.

### Форматы данных

- **Цены, вес, калории** — decimal-строки: `"399.00"`, `"380.00"`. Конвертируйте через `parseFloat()` для арифметики.
- **Даты** — ISO 8601 в UTC: `"2024-01-15T12:00:00Z"`.
- **`size_value` в списке блюд** — строка с единицей: `"25 см"`.
- **`size_value` в детальном варианте** — только число: `"25.00"`, единица отдельно в `measure_unit.short`.
